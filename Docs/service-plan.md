# AI 기반 마케터용 인사이트 캘린더

## 📋 서비스 개요

### 서비스명
**AI 인사이트 캘린더** (AI Insight Calendar for Marketers)

### 한 줄 소개
마케터가 매일 하나의 AI 생성 인사이트를 받아 트렌드를 사고와 기획으로 연결하는 캘린더형 사고 도구

### 핵심 가치 제안
"트렌드 소비"를 "사고 축적"으로 전환하는 일상적 구조 제공

---

## 🎯 문제 정의

### 문제 상황

마케터는 트렌드·자료·사례를 **'많이'** 접한다  
그러나 이를 사고로 전환하는 구조는 거의 없다

**결과적으로:**
- 인사이트는 흘려보내지고
- 기획은 감각·경험 의존도가 높아진다

### 핵심 문제 정의

> **마케터는 트렌드를 소비하지만, 이를 사고와 기획으로 연결할 수 있는 일상적 구조가 부족하다.**

| 구분 | 내용 |
|------|------|
| **문제의 주체** | 마케터 |
| **문제의 원인** | 구조·형식 부재 |
| **문제의 결과** | 사고 축적 불가 |

---

## 💡 해결 방향

### 해결 아이디어

> **AI를 활용해 매일 하나의 인사이트와 트렌드 신호를 제공하는 '캘린더형 사고 도구'**

### 왜 캘린더인가?

#### 1. 매일 확인하는 형식 → 습관화
- 캘린더는 일상에서 자연스럽게 확인하는 도구
- 별도의 '학습' 없이 루틴으로 사용 가능

#### 2. 과도한 정보 제공 방지
- 하루 1개의 인사이트만 제공
- 정보 과부하 없이 깊이 있는 사고 가능

#### 3. 지속성과 아카이빙
- 과거 인사이트를 언제든 다시 확인
- 시간의 흐름에 따른 트렌드 변화 추적 가능

## ⚙️ 핵심 기능

### [기능 1] 데일리 인사이트 생성

**하루 1개의 인사이트 문장 자동 생성**

- AI가 트렌드 데이터를 분석하여 마케팅 관점의 인사이트를 도출
- 간결하고 명확한 한 문장 형태

**예시:**
```
"요즘 브랜드는 혜택보다 선택 이유를 설계한다."
```

---

### [기능 2] 트렌드 키워드 제공

**인사이트와 연결된 트렌드 키워드 2~3개**

- 각 키워드에 짧은 해석 포함
- 인사이트의 근거가 되는 트렌드 제시

**예시:**
- **브랜드 스토리텔링**: 단순 제품 소개를 넘어 브랜드의 가치와 철학 전달
- **선택 피로감**: 너무 많은 옵션보다 명확한 선택 이유 제공이 중요
- **가치 소비**: MZ세대의 소비 기준이 '가성비'에서 '의미'로 이동

---

### [기능 3] 맥락 설명 (Why it matters)

**인사이트 + 트렌드를 연결한 해석 텍스트**

- "왜 지금 중요한가"를 설명
- 마케터가 이 인사이트를 어떻게 활용할 수 있는지 제시

**예시:**
```
최근 소비자들은 제품의 기능이나 할인보다 "왜 이 브랜드를 선택해야 하는가"에 
더 민감하게 반응합니다. 단순한 혜택 경쟁에서 벗어나 브랜드의 가치와 철학을 
명확히 전달하는 것이 차별화의 핵심입니다.
```

---

### [기능 4] 사고 확장 질문 제공

**마케터 관점의 실무 적용 질문 1개**

- 실제 기획에 바로 적용 가능한 질문
- 사고를 확장하고 실행으로 연결하는 브리지 역할

**예시:**
```
💭 "우리 브랜드가 제공하는 '선택 이유'는 명확한가? 
    고객에게 어떤 가치를 전달하고 있는가?"
```

---

### [기능 5] 캘린더 UI & 아카이브

**월간 캘린더 형태로 인사이트 관리**

- 한 눈에 보는 월간 인사이트 타임라인
- 과거 인사이트 다시 열람 가능
- 특정 날짜 클릭 시 해당 인사이트 카드 노출

**사용성:**
- 날짜별 인사이트 검색 및 필터링
- 북마크/즐겨찾기 기능
- 인사이트 축적을 통한 개인화된 트렌드 라이브러리 구축

## ✨ 인사이트 품질 기준

### 좋은 인사이트의 4가지 필수 조건

#### 1. 시의성 (Timely)
- 최근 2주 이내의 트렌드를 반영
- "지금" 중요한 변화를 포착
- 과거형이 아닌 현재 진행형

#### 2. 구체성 (Specific)
- 추상적인 표현 지양
- 명확한 현상이나 행동 패턴 명시
- "요즘", "최근", "지금"의 구체적 변화

#### 3. 근거 기반 (Evidence-based)
- 데이터, 사례, 트렌드에 기반
- 개인 의견이 아닌 관찰 가능한 현상
- 트렌드 키워드로 근거 제시

#### 4. 실행 가능성 (Actionable)
- 마케터가 실무에 바로 적용 가능
- 기획, 전략 수립에 활용 가능한 관점
- "그래서 어떻게 해야 하는가"에 대한 힌트 포함

---

### 나쁜 인사이트 예시

| 나쁜 예시 | 문제점 | 개선 방향 |
|----------|--------|-----------|
| **"AI 기술이 발전하고 있다"** | 너무 일반적, 시의성 부족 | → "마케터들이 AI를 기획 보조가 아닌 실행 도구로 쓰기 시작했다" |
| **"소비자들이 변화하고 있다"** | 추상적, 구체성 없음 | → "Z세대는 제품 리뷰보다 브랜드 뒷이야기에 더 반응한다" |
| **"마케팅이 중요하다"** | 당연한 얘기, 새로움 없음 | → "요즘 브랜드는 광고비를 줄이고 커뮤니티 운영에 투자한다" |

---

### 좋은 인사이트 예시

#### 예시 1: 소비자 행동 패턴
```
"Z세대는 광고를 콘텐츠로 소비한다"
```
- ✅ 구체적 (Z세대, 광고, 콘텐츠)
- ✅ 시의성 (최근 TikTok, Reels 트렌드)
- ✅ 실행 가능 (광고를 콘텐츠로 만들어야 함)

#### 예시 2: 브랜딩 전략
```
"요즘 브랜드는 혜택보다 선택 이유를 설계한다"
```
- ✅ 구체적 (혜택 vs 선택 이유)
- ✅ 시의성 (가치 소비 트렌드)
- ✅ 실행 가능 (브랜드 메시지 방향 전환)

#### 예시 3: 플랫폼 트렌드
```
"짧은 영상이 길어지고, 긴 글이 짧아진다"
```
- ✅ 구체적 (영상/글의 길이 변화)
- ✅ 시의성 (YouTube Shorts 10분 확장, 트위터 축소)
- ✅ 실행 가능 (콘텐츠 포맷 전략 수정)

### 인사이트 주제 카테고리

MVP에서 다루는 마케팅 주제 범위:

| 카테고리 | 설명 | 예시 키워드 |
|---------|------|------------|
| **소비자 행동** | 구매·소비 패턴 변화 | Z세대, 가치 소비, 선택 기준 |
| **플랫폼 트렌드** | SNS, 채널 변화 | TikTok, Reels, 알고리즘 |
| **콘텐츠 전략** | 포맷, 메시지 트렌드 | 숏폼, 스토리텔링, UGC |
| **브랜드 전략** | 브랜딩, 포지셔닝 | 브랜드 정체성, 차별화, 가치 |
| **캠페인 사례** | 성공/실패 사례 분석 | 바이럴, 협업, 이벤트 |
| **마케팅 기술** | 도구, 자동화 트렌드 | AI, 개인화, 데이터 분석 |

**원칙:** 특정 산업군에 한정하지 않고, 모든 마케터에게 적용 가능한 보편적 인사이트 제공

### 품질 검증 프로세스

#### 발행 전 체크리스트

매일 생성된 인사이트가 다음 조건을 충족하는지 확인:

- [ ] 4가지 필수 조건 충족 (시의성/구체성/근거/실행 가능성)
- [ ] 트렌드 키워드 2-3개가 논리적으로 연결됨
- [ ] 맥락 설명이 "왜 중요한가"를 명확히 전달
- [ ] 실무 질문이 인사이트와 직접 연결됨
- [ ] 문장이 간결하고 이해하기 쉬움 (1분 이내 읽기 가능)

#### 품질 미달 시 대응

**원칙:** 품질 없는 발행보다는 발행 연기. 신뢰가 우선.

| 상황 | 대응 방안 |
|------|----------|
| **AI 생성 품질 미달** | 1. 자동 재생성 시도 (최대 3회)<br>2. 백업 에버그린 인사이트 발행<br>3. 또는 과거 인기 인사이트 재해석 버전 |
| **데이터 수집 실패** | 사전 준비된 에버그린 인사이트 활용 |
| **모든 시도 실패** | 발행 연기 + 익일 2개 발행 (보상) |

**백업 콘텐츠 전략:**
- 에버그린 인사이트 20개 사전 준비 (보편적 마케팅 원칙)
- 예: "소비자는 제품이 아니라 결과를 산다"
- 예: "브랜드는 기억보다 반복으로 만들어진다"
- 긴급 시에만 사용, 남발 금지

#### 사용자 피드백 활용

- 👍 긍정 피드백 70% 이상 → 유사 패턴 강화
- 👎 부정 피드백 30% 이상 → 해당 유형 즉시 개선
- 매주 피드백 데이터 분석하여 AI 프롬프트 조정

**개선 원칙:** 
- 발행 후 피드백으로 지속 학습
- 하지만 발행 전 최소 품질 기준은 반드시 확보
- 완벽주의는 경계하되, 사용자 신뢰는 사수

## 🤖 AI 기술 활용 방식

### AI의 역할

**단순 요약 도구 ✗**  
**인사이트 편집자 ✓**

AI 없이는 자동화가 불가능한 영역:
- 방대한 데이터에서 패턴 추출
- 마케팅 맥락으로 재해석
- 실무에 적용 가능한 질문 생성

### AI 작동 구조 (4단계)

```
1단계: 트렌드·키워드 데이터 수집
   ↓ (소셜 미디어, 뉴스, 검색 트렌드 등)

2단계: 반복 패턴 및 공통 맥락 추출
   ↓ (AI 분석을 통한 패턴 인식)

3단계: 마케팅 관점으로 재해석
   ↓ (인사이트 문장 생성)

4단계: 인사이트 문장 + 질문 생성
   ↓ (최종 콘텐츠 제공)
```

## 🏭 콘텐츠 운영 계획

### 운영 방식: 100% AI 자동화

**핵심 전략:** 사람의 개입을 최소화하고 AI가 완전 자동으로 매일 인사이트 생성

#### 왜 완전 자동화인가?

| 장점 | 설명 |
|------|------|
| **확장 가능성** | 사용자가 늘어나도 운영 비용 일정 |
| **일관성** | 매일 빠짐없이 정해진 시간에 발행 |
| **단독 운영 가능** | 개발자 1명이 혼자 운영 가능 |
| **빠른 검증** | 초기 MVP 단계에서 빠르게 테스트 |

#### 리스크와 대응

| 리스크 | 대응 방안 |
|--------|----------|
| AI 품질 불안정 | 사용자 피드백으로 지속 개선 |
| 데이터 소스 오류 | 다중 소스 활용으로 안정성 확보 |
| 긴급 상황 대응 어려움 | 자동 재시도 로직 구현 |

---

### 데이터 수집 소스 (5가지)

#### 1. Google Trends
- **용도:** 검색 트렌드 키워드 분석
- **수집 주기:** 매일 오전 6시
- **활용:** 최근 7일간 급상승 마케팅 관련 키워드

#### 2. 뉴스 API
- **용도:** 마케팅/비즈니스 뉴스 헤드라인
- **소스:** NewsAPI, Google News API
- **수집 주기:** 매일 오전 6시
- **활용:** 주요 뉴스에서 트렌드 신호 포착

#### 3. 마케팅 뉴스레터
- **소스:** 캐릿, 어피티, Marpple, Marketing Brew 등
- **용도:** 전문가가 큐레이션한 마케팅 이슈
- **수집 방법:** RSS 피드 또는 이메일 파싱
- **활용:** 질 높은 마케팅 인사이트 원천

#### 4. Reddit/LinkedIn 커뮤니티
- **소스:** 
  - Reddit: r/marketing, r/socialmedia, r/advertising
  - LinkedIn: 마케팅 그룹 인기 게시글
- **용도:** 실무자들이 논의하는 생생한 이슈
- **수집 주기:** 주간 인기 글 (Upvote 기준)
- **활용:** 현장의 리얼한 고민과 트렌드 파악

#### 5. 사례 플랫폼
- **소스:** 오픈서베이, 브랜드브리프, Ads of the World
- **용도:** 실제 캠페인 사례 및 성과 데이터
- **수집 방법:** 주간 베스트 케이스 크롤링
- **활용:** 성공 사례 기반 인사이트 도출

---

### AI 자동화 파이프라인

```
[매일 오전 6:00] 데이터 수집 시작
   ↓
5개 소스에서 데이터 수집
(Google Trends, 뉴스, 뉴스레터, Reddit/LinkedIn, 사례)
   ↓
[오전 6:10] 데이터 전처리
- 중복 제거
- 마케팅 관련 필터링
- 신뢰도 낮은 데이터 제외
   ↓
[오전 6:20] AI 분석 (LLM)
- 공통 패턴 추출
- 트렌드 신호 탐지
- 인사이트 후보 3개 생성
   ↓
[오전 6:30] 품질 검증 (자동)
- 4가지 필수 조건 체크 (시의성/구체성/근거/실행)
- 가독성 점수 계산
- 최적 1개 선택
   ↓
[오전 6:40] 콘텐츠 생성
- 인사이트 문장 최종 정제
- 트렌드 키워드 2-3개 추출
- 맥락 설명 생성 (200자)
- 실무 질문 생성
   ↓
[오전 6:50] 최종 검증 및 저장
- 논리적 일관성 체크
- DB 저장 (발행 예약)
   ↓
[오전 7:00] 자동 발행
- 사용자에게 노출
```

- 새로운 도구 학습에 거부감이 적음

### UX 설계 원칙

#### 1. 설명 없이 이해 가능
- 직관적인 캘린더 UI

#### 2. 스크롤 최소화
- 하루 1개 콘텐츠로 압축
- 모바일 화면 1~2번 스크롤로 완독 가능

#### 3. 읽기 1분 이내
- 인사이트 문장 + 키워드 + 맥락 + 질문
- 전체 읽는 시간: 약 1분

#### 4. 사용 부담 낮음
- 학습이 아닌 '사고 루틴' 경험
- 매일 부담 없이 확인 가능한 분량

## 🚶 사용자 시나리오

### 시나리오 1: 신규 방문 사용자 (첫날)

**상황:** 주니어 마케터 지민(26세)이 SNS에서 서비스를 알게 됨

**여정:**

1. **서비스 접속** (오후 2시)
   - 랜딩 페이지 도착
   - "매일 1분, 마케터를 위한 인사이트" 문구 확인
   
2. **즉시 인사이트 확인** (회원가입 없음)
   - 오늘의 인사이트 카드가 바로 노출됨
   - "요즘 브랜드는 혜택보다 선택 이유를 설계한다" 읽기
   - 트렌드 키워드 3개 + 맥락 설명 읽기 (약 1분)
   
3. **흥미 유발**
   - 💭 질문: "우리 브랜드의 선택 이유는 명확한가?" 
   - "아, 이거 다음에도 보고 싶은데?" 생각
   
4. **북마크 시도**
   - 북마크 버튼 클릭 시 → 간단한 정보 입력 안내
   - 이메일만 입력 (선택적)
   
5. **재방문 의지 형성**
   - 내일 오전 7시 새 인사이트 안내 확인
   - 서비스 북마크 또는 알림 설정

**결과:** 별도 학습 없이 즉시 가치 경험 → 재방문 동기 생성

---

### 시나리오 2: 일상 사용자 (D+7, 일주일 후)

**상황:** 지민이 일주일째 매일 아침 서비스를 확인 중

**아침 루틴:**

1. **알림 수신** (오전 7시)
   - 푸시 알림: "오늘의 인사이트가 도착했어요 📅"
   
2. **출근 중 확인** (오전 7:30, 지하철)
   - 모바일에서 캘린더 열기
   - 오늘 날짜 클릭 → 인사이트 카드 열람
   - "Z세대는 광고를 콘텐츠로 소비한다" 확인
   
3. **빠른 읽기** (1분)
   - 핵심 인사이트 + 트렌드 키워드 확인
   - 실무 질문 읽고 잠깐 생각해봄
   
4. **북마크 저장**
   - "이거 다음주 캠페인 기획에 쓸 수 있겠다"
   - 별표 아이콘 클릭 → 저장 완료
   
5. **과거 확인** (선택적)
   - 어제 인사이트가 궁금해서 스와이프로 확인
   - 월간 캘린더 뷰에서 한눈에 보기

**결과:** 출근길 1분 루틴으로 자리잡음 → 습관 형성

---

### 시나리오 3: 기획 준비 사용자 (D+30)

**상황:** 지민이 신규 브랜딩 캠페인 제안서를 준비 중

**활용 여정:**

1. **기획 아이디어 필요** (오후 3시, 사무실)
   - "최근 소비 트렌드 관련 인사이트가 뭐 있었지?"
   - 서비스 접속
   
2. **아카이브 탐색**
   - 캘린더 뷰에서 지난 30일 인사이트 훑어봄
   - 북마크한 항목만 필터링
   
3. **관련 인사이트 재확인**
   - "브랜드는 선택 이유를 설계한다" (2주 전)
   - "가치 소비가 주류가 되고 있다" (1주 전)
   - "스토리텔링이 제품 스펙을 이긴다" (3일 전)
   
4. **제안서에 활용**
   - 3개 인사이트를 바탕으로 전략 방향 수립
   - 각 인사이트의 트렌드 키워드를 근거로 활용
   
5. **추가 사고 확장**
   - 인사이트에 딸린 질문들을 제안서 구조에 반영
   - "우리 브랜드의 스토리는 무엇인가?"

**결과:** 흩어진 정보가 아닌 체계적으로 축적된 사고 자산 활용

---

### 핵심 지표: DAU/MAU 비율

**정의:** Daily Active Users / Monthly Active Users

**의미:** 
- 월간 사용자 중 매일 사용하는 사용자 비율
- 습관화 및 서비스 중독성(Stickiness) 측정
- 높을수록 일상적으로 사용한다는 의미

**목표 수치 (1개월 후):**

| 단계 | DAU/MAU 비율 | 의미 |
|------|-------------|------|
| **최소 목표** | 15% | 월간 사용자가 주 1회 이상 방문 |
| **적정 목표** | 25% | 주 2-3회 방문 (루틴 형성 중) |
| **이상적 목표** | 40% | 거의 매일 방문 (완전 습관화) |

**현실적 판단:**
- 1개월차: 15% 달성 시 → 계속 진행
- 1개월차: 10% 미만 → 콘텐츠 품질 또는 UX 문제 점검

  - 피드백이 긍정적으로 변하는가?

**상황 보며 결정:**
- 예상보다 빠르게 성장 → 목표 상향 조정
- 예상보다 느리지만 방향성 좋음 → 인내심 유지
- 방향성도 나쁨 → 빠른 개선 또는 피봇

## 🚀 플랫폼 및 실행 전략

### 플랫폼: 모바일 웹 우선 (PWA)

**선택 이유:**

| 이유 | 설명 |
|------|------|
| **타깃 사용 패턴** | 출퇴근 시간, 이동 중 확인 → 모바일 최적화 필수 |
| **빠른 출시** | 네이티브 앱보다 개발 기간 짧음 (1/3 수준) |
| **설치 장벽 낮음** | 앱스토어 없이 바로 사용 + 홈 화면 추가 가능 |
| **크로스 플랫폼** | iOS/Android 동시 지원, 데스크탑도 자동 대응 |
| **업데이트 용이** | 배포 즉시 모든 사용자에게 반영 |

**PWA 핵심 기능:**
- 홈 화면 추가 (Add to Home Screen)
- 푸시 알림 (오전 7시 인사이트 도착)
- 오프라인 캐싱 (최근 7일 인사이트)
- 빠른 로딩 (1초 이내)

**데스크탑 대응:**
- 반응형 디자인으로 자동 대응
- 하지만 UX는 모바일 중심으로 설계
- 데스크탑은 "보너스" 경험

---

### 회원가입: 불필요 (완전 공개)

**전략:** 누구나 즉시 모든 콘텐츠 확인 가능

#### 장점

| 장점 | 효과 |
|------|------|
| **진입 장벽 제로** | 클릭 1번으로 인사이트 확인 |
| **빠른 가치 체험** | 5초 만에 서비스 가치 전달 |
| **바이럴 용이** | URL 공유만으로 확산 가능 |
| **데이터 수집 불필요** | 개인정보보호 이슈 없음 |

#### 단점과 대응

| 단점 | 대응 방안 |
|------|----------|
| 사용자 데이터 수집 어려움 | 브라우저 로컬 스토리지 활용 (북마크 등) |
| 리텐션 추적 제한 | 익명 세션 ID로 행동 패턴 분석 |
| 알림 발송 어려움 | 선택적 알림 구독 (이메일/푸시) |

#### 선택적 정보 입력

사용자가 원할 때만 최소 정보 입력:

| 기능 | 필요 정보 | 시점 |
|------|----------|------|
| **북마크 저장** | 없음 | 로컬 저장 |
| **푸시 알림** | 알림 권한 | 알림 설정 시 |
| **이메일 알림** | 이메일 | 이메일 구독 시 |
| **기기 간 동기화** | 선택 (향후) | 요청 시 |

**철학:** "가치 먼저, 정보는 나중에"

### 초기 사용자 확보: 소셜 유기적 확산

**전략:** 광고비 0원으로 시작, 콘텐츠 품질로 승부

#### 확산 채널

**1. 개인 SNS (1주차)**
- 개발자 개인 트위터, LinkedIn, 브런치
- "매일 마케팅 인사이트 받아보세요" 소개
- 첫 일주일 인사이트 샘플 공유

**2. 마케팅 커뮤니티 (2주차)**
- Reddit r/marketing
- LinkedIn 마케팅 그룹
- 국내: 오픈 카카오톡 마케팅방, 페이스북 그룹
- 접근: "제가 만든 서비스 피드백 부탁드립니다" (겸손)

**3. 콘텐츠 마케팅 (3-4주차)**
- 인기 있었던 인사이트 TOP 5 정리글
- "이번 주 마케팅 트렌드 요약" 포맷
- 브런치, Medium, LinkedIn 게시
- 마지막에 "더 보려면 ○○" 자연스러운 유입

**4. 입소문 (지속)**
- 사용자가 인사이트를 스크린샷으로 공유하도록 유도
- 이미지에 서비스 워터마크 (작게)
- "출처: AI 인사이트 캘린더"

#### 목표 유입 경로 비율 (1개월)

```
개인 네트워크: 40%
커뮤니티: 30%
콘텐츠/검색: 20%
직접 재방문: 10%
```

#### 하지 않을 것

- ❌ 초기 유료 광고 (ROI 불확실)
- ❌ 인플루언서 협찬 (비용 대비 효과 낮음)
- ❌ 스팸성 홍보 (브랜드 이미지 손상)

### 수익화: PMF 확인 후 고려

**현재 단계:** 수익 모델 없음, 완전 무료

#### PMF(Product-Market Fit) 판단 기준

다음 조건 충족 시 수익화 고려:

- [ ] DAU/MAU 비율 30% 이상 (강한 습관화)
- [ ] 누적 사용자 1,000명 이상
- [ ] 사용자가 "이거 없으면 불편하다" 피드백
- [ ] 자발적 후원 제안 or "유료로 전환해도 쓸 의향" 확인

**예상 시점:** 빠르면 3개월, 보통 6개월 후

#### 수익화 옵션 (미래)

| 모델 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **프리미엄** | 기본 무료, 추가 기능 유료 | 무료 사용자 유지 | 기능 분리 어려움 |
| **구독** | 월 $5-10 정액제 | 예측 가능한 수익 | 진입 장벽 생김 |
| **후원** | Buy Me a Coffee | 부담 없음 | 수익 불안정 |
| **B2B** | 기업용 팀 플랜 | 높은 단가 | 기능 확장 필요 |

**현재 방침:** PMF 전에는 수익화 고려 안 함. 사용자 습관화가 우선.

### 경쟁 포지셔닝: 새로운 카테고리 창출

**전략:** 기존 서비스와 비교하지 않고, 새로운 카테고리 정의

#### 기존 서비스들

| 카테고리 | 서비스 예시 | 한계 |
|---------|------------|------|
| 캘린더 도구 | Google Calendar, Notion Calendar | 일정 관리용, 인사이트 없음 |
| 뉴스레터 | 캐릿, 어피티 | 주간 발행, 정보 과다 |
| AI 도구 | ChatGPT | 능동적 질문 필요, 루틴 아님 |
| 마케팅 툴 | Hootsuite, Buffer | 실행 도구, 사고 도구 아님 |

#### 우리의 독보적 포지셔닝

```
"AI 인사이트 캘린더는 마케터를 위한 
매일 1분 사고 루틴입니다"
```

**차별점:**
- ✅ 캘린더 형식 (일정 아닌 인사이트 관리)
- ✅ 매일 1개 (정보 과잉 방지)
- ✅ AI 자동 생성 (사람 큐레이션 아님)
- ✅ 사고 확장 질문 (단순 정보 전달 아님)
- ✅ 루틴 중심 (찾아가는 게 아닌 받아보는)

**메시지:**
- "뉴스레터는 일주일에 한 번, 우리는 매일"
- "캘린더는 일정을 관리하지만, 우리는 생각을 관리합니다"
- "ChatGPT는 질문해야 하지만, 우리는 먼저 알려줍니다"
